Scénario de déploiement pour une gestion par projet

Ce document décrit quelques lignes directrices pour un déploiement progressif d'odoo 13 community dans une petite ou moyenne entreprise/industrie (PME/PMI) dont la gestion est orientée Projets.

Il est structuré sous forme d'activités d'apprentissage destinées aux utilisateurs finaux et dédiées à chaque module.

Prérequis​

  • une installation propre d'odoo ERP. Pour un deployment avec docker consulter setup odoo on docker.
  • préconfiguration et personnalisation d'odoo selon les spécificités de l'entreprise, particulièrement la création des comptes utilisateurs pour chaque apprenant. Pour plus de détails consulter la section configuration.

Hypothèses​

  • Petite ou Moyenne Entreprise/Industrie (PME/PMI) de 20 personnes.
  • Le déploiement final se feait de manière progressive (par module) avec une durrée d'absorption d'au moins deux (02) semaines.
  • Groupes de 8 à 10 apprenants ayant chacun mis à jour son profil utilisateur.
  • Processus métier simples et maturés. Si les processus n'existent pas il faudra les crées.
  • Outils de collaboration : tableau blanc, post-it, marqueurs, ...
  • Equipe projet composé d'un chef de projet et de membres ayant une autorité suffisante sur les activités qui seront implémentées.

Module Messages​

Les activités d'apprentissage du Module Messages sont :

  1. lister les départements / fonctions / projets existants dans l'entreprise et constituer des groupes dont les conversations sont privés selon la liste ci-dessus.
  2. définir les interactions entre les différents groupes.
  3. définir des sujets de discussions directes entre deux (02) interloculteurs.
  4. définir les types d'informations dont la diffusion est générale à tous les employés.

Pour chaque activité, créer les cannaux correspondants et produire du contenu (texte, emojis, fichiers attachés, ...).

Module Employés​

Les activités d'apprentissage du Module Employés sont :

  1. identifier la hiérarchie des structures existantes dans l'entreprise et les postes de travail.
  2. créer les départements.
  3. créer les postes de travail.
  4. créer des employés gestionnaires et les associés aux départements.
  5. créer des employés collaborateurs.
  6. associés des employés à des comptes utilisateurs et inviter les à se connecter.

Retour au sommaire

Module Vente​

Les activités d'apprentissage du Module Vente sont :

  1. identifier sur papier puis mettre à jours les paramètres suivants :
    1. caractéristiques principales des articles vendus par l'entreprise ainsi que les éventuelles variantes.
    2. créer des comptes utilisateurs pour les commerciaux de l'entreprise, puis intégrer les dans des équipes commerciales.
  2. selon le cas, renseigner au moins 10 articles vendus par l'entreprise, ainsi que 10 clients.
  3. créer au moins 05 devis avec toutes les informations nécessaires y compris des articles optionnels. Imprimer-les et envoyer-les par mail puis Confirmer-les.
  4. créer des factures pour les bon de commandes.

Module Projet - activités de base​

Les activités de base d'apprentissage du Module Projet sont :

  1. identifier sur papier puis mettre à jours les paramètres suivants :
    1. Types d'activités.
    2. Etiquettes des taches.
  2. si nécessaire, créer des employés pour les membres des équipes projets de l'entreprise et leur associés des utilisateurs.
  3. si nécessaire, renseigner au moins 05 clients.
  4. créer au moins 02 projets et renseigner les informations y relatives.
  5. ajouter des pièces jointes de différents formats aux projets.
  6. dans la partie Projets sélection un des projets :
    1. ajouter des colonnes (TODO, WIP, DONE).
    2. paramétrer chaque colone.
  7. ajouter des tâches dans chaque étape du projets.
  8. editer les tâches et renseigner les informations y relatives.
  9. faite glisser les taches entre les différentes étapes et constater les changments dans la partie analyse.
  10. modifier l'état des tâches et constater le indicateurs couleurs sur le kanban projet et dans la partie Toutes les taches.

Module projet - activités avancées​

Les activités d'apprentissage avancées de ce module sont :

  1. identifier sur papier les éléments suivants :

    1. projets en cours au niveau de l'entreprise.
    2. tâches relatives à chaque projet.
    3. sous-taches des taches listées ci-avant.

  2. avec l'option Utiliser les évaluations de projet activé dans la configuration :

    1. cette option nécessite des informations client fiables, ainsi que la configuration d'un serveur de messagerie et/ou l'accès client au portail de l'entreprise.
    2. pour l'un des projets identifiés, définir un Modèle d'email pour les évaluations pour chaque étape.
    3. le client recevra un email de demande d'evaluation chaque fois q'une tâche progresse dans le pipeline et les feedbacks seront visibles directement dans la vue kanban du projets avec chaque tâche.

  3. avec l'option Sous-tâches activé dans la configuration :

    1. renseigner les informations relatives à un des projets identifiés.
    2. renseigner les tâches du projet.
    3. créer les sous-tâches en les reliants à leurs taches parentes.
    4. assigner les taches/sous-tâches à des membres de l'équipe.
    5. chaque membre constate la liste des tâches qui lui sont assignés et les fait progrésser dans le pipeline.
    6. refaire l'opération pour les autres projets identifiés.
    7. créer une hiérarchie de projets en les reliants avec le champ Sous-tache du projet.

  4. avec l'option Feuilles de temps sur les tâches activé dans la configuration.

    1. pour l'un des projets identifiés, activé l'option Feuilles de temps.
    2. créer un nouveau modèle de Temps de travail adapté à la législation locale (Heures de travail, Congé Global).
    3. renseigner le Coût de la feuille de temps pour chaque membre de l'équipe (module employés).
    4. pour chaque tâche du projet selectionnée, renseigner le champ Heures planifiées.
    5. les membres d'équipes peuvent maintenant renseigner leurs apports en volume horaire à chaque tâche et visualiser les heures restantes dans la vue kanban du projet.
    6. visualiser les changements dans la partie analyse après chaque modification sur les horaires travailés par employé / par tache / par projet.
    7. dans le module Feuilles de temps :
      1. chaque membre de l'équipe visulise ses feuilles de temps.
      2. visualiser la liste de toutes les feuilles de temps.
      3. faite des analyses par employé / par tache / par projet.
      4. changer l'option Unité d'encodage dans la configuration et visualiser les modifications apportées aux informations projets / taches / feuilles de temps du module Projet.
    8. refaire l'opération pour les autres projets identifiés.

Intégration modules Projet avec Vente​

NB : Il est primordiale de créer des lignes de ventes (artciles de type service) indépendantes des aspects techniques, et de gérer ces derniers (aspects techniques) comme des tâches dans le projet.

Workflow 1 : Bon de Commandes à partir d'un projet

  1. dans le module Vente créer des articles de type service et configurer la Politique de facturation du service sur Feuilles de temps sur les tâches. et Suivi du service sur Ne pas créer la tâche.
  2. pour l'un des projets identifiés, activé l'option Feuilles de temps.
  3. lancer la commande Créer un bon de commande.
  4. si le type de facturation est définie sur Au taux du projet.
    1. selectionner un des artciles de type service.
    2. un bon de commande est crée avec comme référence le projet en cours.
    3. on constate qu'une seule ligne de commande est ajoutée, celle du service séléctionné.
  5. si le type de facturation est définie sur Au taux de l'employé.
    1. ajouter des lignes employé / service.
    2. un employé ne peut être séléctionner qu'une seule fois, çad qu'il ne peut contribuer qu'à un seul service.
    3. un bon de commande est crée avec comme référence le projet en cours.
    4. on constate que plusieurs lignes de commande sont ajoutées, chacune pour correspondant à un des services séléctionnés.
  6. retour dans le module projet.
  7. sélectionner quelques tâches du projet en cours et ajouter des feuilles de temps, pour faire progrésser le taux d'avancement.
  8. dans les paramètres du projet en cours, onglet Facturation, sélectionner le lien du Bon de commande correspondant.
  9. les heures (feuilles de temps) réalisées dans le projet sont automatiquement renseigné dans la colonne Livré.
  10. lancer la commande Créer une facture, sélectionner Facture normale.
  11. une nouvelle facture est crée pour le projet / bon de commande.
  12. revenir aux paramètres du Bon de commande. on constate que la quanitié facturée est mise à jour.
  13. si on vérifie les feuilles de temps enregistrées dans la Tâches, on constate qu'elles sont grisées et impossible à modifier puisqu'elles ont déjà été facturées.
  14. revenir aux paramètres du Bon de commande du projet. lancer la commande Vue d'ensemble du projet, on obtient un rapport détaillant les Heures encodées, la Profitabilité, les feuilles de temps par employé, ...
  15. aussi, dans la page des paramètres du Bon de commande du projet, lancer la commande Aperçu client, on obtient un rapport mise en forme tel qu'il sera visible par le client.
  16. pour facturer de nouveau, il faute faire progrésser le projet en renseignant de nouvelles lignes de feuilles de temps dans les paramètres des Tâches. ainsi, à chaque jalon du projet, une facture particulière peut être générée.
  17. pour associer plusieurs tâches techniques à une même ligne de vente (cas général), créer une nouvelle tache dans le projet. dans l'onglet infos supplémentaires de la tâche sélectionner la Ligne de vente désirée. ajouter des feuilles de temps à cette tâche.
  18. revenir aux paramètres du Bon de commande. on constate que la quantité livrée est automatiquement mise à jours comme étant la somme des heurs réalisées sur l'ensemble des tâches liées à la ligne de vente.

Workflow 2 : Service (article) spécifique à un seul Projet

  1. pour l'un des projets identifiés, activé l'option Feuilles de temps.
  2. dans le module Vente créer des articles de type service et configurer la Politique de facturation du service sur Feuilles de temps sur les tâches. et Suivi du service sur Créer une tâche dans un projet existant.
  3. créer un nouveau Devis et insérer l'article configuré précédement. on constate que le devis est automatiquement lié au projet référencé dans les paramètres de l'article (service).
  4. confirmer le Devis pour le transformer en Bon de commande.
  5. dans le module projet, on constate qu'une nouvelle tache est automatiquement ajoutée au projet en cours et la ligne de vente fait référence au bon de commande crée.
  6. la facturation est effectuée de la même manière que dans le workflow précédent (ajout de feuilles de temps dans les tâches -> création d'une facture -> aprçu).
  7. pour associer plusieurs tâches techniques à cette ligne de vente, créer une nouvelle tache dans le projet. dans l'onglet infos supplémentaires de la tâche, définir la tâche parent sur la tâche liée à la ligne de vente. on constate que la ligne de vente de la sous-tâche est automatiquement définie sur la ligne de vente de la tache parente. ajouter des feuilles de temps à cette sous-tâche.
  8. revenir aux paramètres du Bon de commande. on constate que la quantité livrée est automatiquement mise à jours comme étant la somme des heurs réalisées sur l'ensemble des tâches liées à la ligne de vente.

Workflow 3 : Projet à partir d'un ou plusieurs Bon de commande

  1. dans le module Vente créer des articles de type service et configurer la Politique de facturation du service sur Feuilles de temps sur les tâches. et Suivi du service sur Créer une tâche dans un projet de bon de commande.
  2. créer un nouveau Devis et insérer l'article configuré précédement. dans l'onglet Autres informations, sélectionner le projet souhaité. Confirmer le Devis (transformer en Bon de commande).
  3. si dans l'étape précédente le projet est laissé vide, un nouveau projet avec la référence du Bon de commande sera crée (situation indésirable).
  4. dans le module projet, on constate qu'une nouvelle tache est automatiquement ajoutée au projet en cours et la ligne de vente fait référence au bon de commande crée.
  5. la facturation est effectuée de la même manière que dans les workflows précédents (ajout de tâches/sous-tâches techniques -> ajout de feuilles de temps -> création d'une facture -> aprçu).

Workflow 4 : Service (article) non lié automatiquement à une seule tâche

  1. dans le module Vente créer des articles de type service et configurer la Politique de facturation du service sur Feuilles de temps sur les tâches. et Suivi du service sur Créer un nouveau projet sans tâche (çad Devis).
  2. créer un nouveau Devis et insérer l'article configuré précédement. Confirmer le Devis (transformer en Bon de commande).
  3. un nouveau projet avec la référence du Bon de commande sera crée sans aucune tâche.
  4. toutes les nouvelles tâches créer dans ce projet seront par défaut liées à la ligne de vente définie lors de la création du bon de commande. bien sûre le gestionnaire de projet peut les lignes de ventes à tout moment.
  5. la facturation est effectuée de la même manière que dans les workflows précédents (ajout de tâches/sous-tâches techniques -> ajout de feuilles de temps -> création d'une facture -> aprçu).

Workflow X : Devis et Projets crées séparément

  1. dans le module Vente créer des articles de type service et configurer la Politique de facturation du service sur Feuilles de temps sur les tâches. et Suivi du service sur Ne pas créer la tâche.
  2. créer un nouveau Devis et insérer le sevice (article) configuré précédement.
  3. confirmer le Devis pour le transformer en Bon de commande.
  4. dans le module projet, crée un nouveau projet ou sélectionner un projet existant. s'assurer que l'option Feuilles de temps est activé.
  5. créer une nouvelle tâche. dans l'onglet Info supplémentaires séléctionner la société pour laquelle le Devis a été crée.
  6. un nouveau champ Ligne de vente apparait. séléctionner le service (article) précédement définit.
  7. la facturation est effectuée de la même manière que dans les workflows précédents (ajout de tâches/sous-tâches techniques -> ajout de feuilles de temps -> création d'une facture -> aprçu).
  8. maintenant vous pouver ajouter autant de services au bon de commande et y lier autant de tâches que nécessaires :).

Module Additionnel Scrum​

Les activités de base d'apprentissage du Module Projet Scrum sont :

  1. identifier sur papier puis mettre à jours les paramètres suivants :
    1. Les projets gérés par la méthode Scrum.
    2. Les périodes durant lesquelles les Sprints sont exécutés et les éventuelles périodes Breath.
    3. Les Equipes Scrum impliquées.
  2. créer un nouevau projet et lui associer un équipe Scrum.
  3. créer des tâches dans le projet et associées les aux sprints.
